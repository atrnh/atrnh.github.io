Software & Tools

Terms you should know that are used everywhere

package manager

package manager

Installing stuff is hard

  • Modern programming languages come with a package manager by default
  • User-created package managers exist for older langs
  • Also there are package managers for your OS
    • Install software via the command line!

package managers by OS

macOS

Homebrew

Follow instructions on homepage to install

Ubuntu (Vagrant’s OS)

apt

Already installed by default

Windows

chocolatey

package managers by language

Node

Uses npm by default

  • Installs stuff in node_modules/
  • Automatically generates list of dependencies in package.json

Many people prefer an alternative package manager, Yarn

Python

pip by default

  • Gotta create requirements.txt by hand w/ pip freeze
  • Pipenv is a great alternative, works more like npm/yarn

Ruby

gem

  • Automatically creates list of dependencies in Gemfile

plugin manager

plugin manager

Like package manager but for installing add-ons & extensions for specific application

Btw, most text editors have plugin managers &
useful extensions

Language-specific tools

dotfile

dotfile

Files used to store system configuration/settings.

They’re usually secret files,
so they always start with . (aka a dot)

  • .bash_profile
  • .zshrc
  • .config/

config file

config file

Any file used to store configuration in general

Usually a .json, .yml, .ini file (or similar)

There are better terminal apps out there!

Languages to learn next

Rust

  • Low-level language, maintained by Mozilla (the Firefox people)
  • Great next language because you can get your hands into how memory is managed
  • They have an official beginner’s textbook that is. so. good.
    • Super beginner-friendly, really great diagrams about computer memory

Great project to make is a command line application (they have an official book for that as well!)

Elixir

  • Functional language, open source, becoming a popular language in webdev
  • Great next language because it will make you understand recursion so much better than Python can
  • I learned off the beginners’ guide in their offical docs just fine

Great project to make is a web app using an Elixir web framework called Phoenix
